# 线上问题定位
通用步骤：
1. 查看具体错误和位置
2. 查看对应日志
3. 寻找异常堆栈
4. 定位代码行
5. 看caused by根因

## 普通异常
1. 查看日志：错误日志，控制台/容器日志，框架日志等
2. 抓取核心关键字：`Exception,Error,Caused By,at xxx.java`
   - 优先从**最顶部异常**->**根异常**(`Caused By`)
3. 异常堆栈阅读+代码定位：通过`类名+行号`确定具体位置

## 栈溢出StackOverflowError
1. **找进程** `ps -ef|grep java1`
2. 导出**线程栈** `jstack <pid> > stack.log`
3. **搜索异常** `StackOverflowError`(`java.lang.StackOverflowError`)
   1. 找重复最多的一行
   2. 看方法具体逻辑
      - 自调无限递归
      - 相互调用，死循环
      - 深度太深
4. 解决
   - 加终止条件
   - 打破循环依赖
   - 深度遍历加限制
   - 递归改循环

## 堆溢出OutOfMemoryError
1. **禁止高危操作**: `jmap -dump:full / jmap -F`(强STW,卡死)
2. 定位进程PID：`ps -ef|grep java`
3. 安全抓取**堆快照**：`jmap -dump:format=b,file=heap.hprof <pid>`
4. 查看**JVM内存概况** `jmap -heap <pid>`
5. **OOM分析**定位(hprof)
   - 工具：MAT,JProfile，JvisualVM
   - 高频堆溢出原因
     - 静态集合泄漏
     - 循环引用、缓存不失效
     - 大对象：一次加载百万数据、超大文件
     - 循环创建对象，循环new
     - 第三方工具(缓存 redis等) 大发序列化
6. 配合jstat预判OOM: `jstat -gc <pid> 1000`
   - 老年代持续上升、GC几乎不下降
   - 频繁Full GC(100%内存泄漏)

**处理流程(企业级最优)**：
1. 先限流/降级/关闭耗时内存接口
2. 临时关闭泄漏定时任务/批量任务
3. 抓取快照hprof+分析泄漏源
4. 小范围热修复/代码堵漏
5. 修复重启


**解决**
1. 线上临时救急(适用于非泄漏情况，如瞬时业务量大、临时大对象或人为配置错误情况)
   - 调大堆内存 `-Xms4g -Xmx4g`
   - 清空全局缓存、临时清内存集合
   - 限流、关停大流量接口
2. 根治方案
   - 销毁长生命周期对象、即使清空集合
   - 缓存加过期、LRU限制大小
   - 大数据分批处理、流式读取、避免大对象
   - 禁用静态集合存业务数据
   - 代码加内存监控、大对象拦截


# 规范
* 命名
  * 对象集合+List(如userList)
  * 对象属性集合+属性s(如userIds)
  * 抽象类加Abstract/Base前缀或后缀
  * 实现类加Impl后缀
* Object
  * 读取数据前判空处理
* 数组/集合
  * 判空处理
  * 索引边界限定：避免索引溢出
  * 遍历优先适用foreach
  * 禁止在循环中删除元素(使用迭代器Iterateor)
* 控制语句
  * 不推荐简化单行代码的代码块
* 异常
  * 分别捕获所有异常
  * 禁止空catch块
  * 禁止捕获Exception所有异常
  * 底层异常向上抛出
  * 自定义异常禁止返回错误码
  * 禁止用异常处理业务逻辑
  * 关闭流/连接等资源必须在finally中


## 底层异常向上抛
      
      以MVC框架为例

1. DAO / Mapper 层（最底层）：
   - 数据库异常、IO 异常、SQL 异常、连接异常……完全不捕获
   - 直接抛出原生受检 / 运行时异常，向上抛给 Service
   - 绝不写 try-catch 吞异常
2. Service 层（业务核心层）
   - 系统底层异常（不可控）: 不捕获、不处理、直接向上抛给 Controller
   - 业务可控异常（主动校验）:手动抛自定义业务异常，不静默、不返回 null
   - 需要捕获场景：需要降级、重试、补偿、特殊业务容错场景
3. Controller 层（接口入口）
   - 禁止在 Controller 写业务 try-catch 兜底
   - 只做：参数接收、简单参数校验、调用 service
   - 所有异常（底层异常 + 业务异常）全部外抛，交给全局异常处理器
4. 所有异常（底层异常 + 业务异常）全部外抛，交给全局异常处理器
   - 捕获自定义业务异常 → 返回友好业务提示
   - 捕获系统底层异常（SQL、NPE、IO）→ 返回统一系统错误
   - 统一日志打印、统一响应格式、统一错误码
